## What's new

Catsnip captures a screenshot **inside or outside the browser**, then edits it — all locally,
with nothing leaving your device.

- **Four capture modes**
  - **Region** — drag a marquee anywhere on the page (also via the `Ctrl/Cmd+Shift+S` shortcut).
  - **Visible area** — the current viewport in one shot.
  - **Full page** — scroll-and-stitch the whole page, including pages that scroll inside an
    inner container (SPAs/app shells).
  - **Screen or window** — capture any display or app window via the OS picker, even with the
    browser minimized.
- **Annotate** — arrow, rectangle, ellipse, freehand pen, highlighter, text, and **blur/redact**,
  with color + size and full **undo/redo**.
- **Editable elements** — a **Select** tool to move, resize (handles), restyle, and delete any
  annotation; double-click text to edit it in place.
- **Inline text** — type directly on the screenshot (no popups).
- **Background frame** — drop the snip on a centered background with rounded corners + shadow;
  gradient presets, solid, transparent, or a custom color; adjustable padding/radius.
- **Crop** to exactly what you want.
- **Text recognition (OCR), fully offline** — **Grab text** mode overlays the recognized text so
  you can drag-select and copy it straight off the image (like Live Text); **Extract text** dumps
  it all to a side panel.
- **Session gallery**, **copy to clipboard**, **save PNG/JPEG**, and **Save all as ZIP**.

## Under the hood — shipped files

| File | Role |
| --- | --- |
| `manifest.json` | MV3 manifest (action popup, background worker, `snip-region` command, `wasm-unsafe-eval` CSP for the bundled OCR). |
| `popup.html/.css/.js` | Mode launcher. |
| `background.js` | Service worker: region/visible/full-page capture engines; hands the snip to the editor via IndexedDB. |
| `overlay.js/.css` | In-page region-selection marquee (injected on demand). |
| `editor.html/.css/.js` | Editor tab: crop / annotate / Grab-text + Extract-text (OCR) / gallery / copy / save / ZIP. |
| `lib/jszip.min.js` | Vendored ZIP library (Save all). |
| `lib/tesseract/*` | Vendored Tesseract OCR engine + English model (loaded only from local URLs). |
| `icons/icon{16,32,48,128}.png` | Toolbar/store icons. |

Dev/build files (`build.sh`, `fetch-ocr.sh`, `icons/gen_icons.py`, docs) are excluded from the zip.

## Permission justifications (paste-ready for the dashboard)

| Permission | Why Catsnip needs it |
| --- | --- |
| `activeTab` | Capture and (for region) overlay the **current tab only**, after you explicitly invoke Catsnip (toolbar click or shortcut). This is what keeps host access minimal — there is no broad `host_permissions`. |
| `tabs` | Read the active tab's URL/title (to name the snip and refuse restricted pages) and open the editor tab. |
| `scripting` | Inject the region-selection overlay and measure/scroll the page for full-page capture, on the active tab. |
| `desktopCapture` | The "Screen or window" mode — let you pick a display or app window to capture (outside the browser). |
| `downloads` | Save the finished image (PNG/JPEG) or ZIP to your Downloads folder. |
| `storage` | Remember your preferences only (annotation color/size, export format, background-frame settings). No screenshot pixels are stored. |
| `clipboardWrite` | "Copy" the image (and copied OCR text) to the clipboard. |

**Single purpose:** "Capture a screenshot of a screen region, browser tab, full page, or any
app window, then crop, annotate, OCR, copy, and save it — entirely on the user's device."

## Chrome Web Store submission checklist

1. **Build the package** (include OCR so it works out of the box):
   ```bash
   ./fetch-ocr.sh        # vendors lib/tesseract/* (~11MB) for offline OCR
   ./build.sh            # -> dist/catsnip-v1.0.0.zip
   ```
   Sanity-check the `unzip -l` output: `manifest.json` at the **root**, no dev files, and
   `lib/tesseract/*` present (so OCR ships).
2. **Create the item** at the [Developer Dashboard](https://chrome.google.com/webstore/devconsole)
   and **upload `dist/catsnip-v1.0.0.zip`**.
3. **Store listing**
   - Name: **Catsnip**. Summary (≤132 chars): *"Snip any region — inside or outside the browser —
     then crop, annotate, OCR, copy & save. Everything stays on your device."*
   - Description: adapt **What's new** above.
   - Category: **Productivity** (or Workflow & Planning). Language: English.
   - Icon: `icons/icon128.png`. Screenshots: 1280×800 (or 640×400) — show region capture, the
     editor with annotations + background frame, and Grab-text selecting text.
4. **Privacy practices tab**
   - **Single purpose**: paste the statement above.
   - **Permission justifications**: paste each row from the table above.
   - **Remote code**: **No** — the extension does not use remote code; JSZip and Tesseract are
     bundled in the package and loaded via `chrome.runtime.getURL`. (If asked about the
     `wasm-unsafe-eval` CSP: it's only to run the **bundled** OCR WebAssembly locally, never
     remote code.)
   - **Data usage**: declare that Catsnip does **not** collect or transmit any user data. Check
     the three certifications (not sold to third parties; not used/transferred for purposes
     unrelated to the single purpose; not used to determine creditworthiness/lending).
   - **Privacy policy URL**: paste
     **https://gist.github.com/artfred16/9a1248ae8f1c5c927cb565d5cd06a625**
     (canonical source: `docs/PRIVACY.md`; update the gist when that file changes).

## Verification (run before submitting)

Load the unpacked extension (`chrome://extensions` → Developer mode → Load unpacked) and confirm:

1. **Region / Visible / Full page** all capture and open the editor; full page stitches a long
   article **and** a page that scrolls inside an inner container.
2. **Screen or window** captures a chosen display/app (with the browser window minimized).
3. **Annotate** (all tools), **Select** (move/resize/restyle/delete), **inline text** (type +
   double-click edit), **crop**, and **undo/redo** behave.
4. **Background frame** — presets, custom color, padding/radius, transparent, off.
5. **Grab text** selects/copies text off the image; **Extract text** fills the panel; with
   `lib/tesseract/` present OCR runs offline (DevTools Network shows **no** requests).
6. **Copy** (clipboard), **Save** (PNG + JPEG), **Save all** (ZIP).
7. **Restricted pages** (`chrome://`, the Web Store) disable the page modes with a clear message;
   "Screen or window" still works.
8. **No network**: with DevTools open on the editor and the worker, exercise every feature and
   confirm zero outbound requests.

## Known limitations

- Full-page capture is paced under Chrome's `captureVisibleTab` rate limit and capped at 16384px
  / a memory budget; extremely tall pages are truncated (and flagged). Pages with virtualized
  lists or nested/transformed scrollers may not capture in full.
- OCR uses the English `tessdata_fast` model (speed over maximum accuracy); other languages aren't
  bundled.
- "Screen or window" requires the OS screen-capture picker and permission; it can't run when the
  browser has no window open.
- Browser/system pages (`chrome://`, the Web Store, other extensions, PDFs) can't be captured
  in-browser — use "Screen or window" instead.
